
# 《六方对弈：弹珠碰撞战》游戏策划案
## 一、游戏概述

### 1.1 游戏名称
**六色弹珠：六边形碰撞战**

### 1.2 游戏类型
回合制策略对战游戏（双人异步对战 ）

### 1.3 游戏平台
PC（后续可扩展至移动端）

### 1.4 核心玩法
玩家在六边形棋盘上操控六种拥有独特能力的弹珠，通过**弹性碰撞**、**角色技能**与**战术布局**，将对方所有弹珠击出棋盘或使其死亡，从而获得胜利。

### 1.5 游戏特色
- **六边形棋盘**：169个格点，策略性强
- **六种弹珠**：白、蓝、绿、红、黑、黄，各具独特机制
- **弹性碰撞系统**：真实感速度交换，战斗节奏紧凑
- **死亡增益机制**：黄球死亡可为友方提供永久强化
- **白球变色机制**：可继承死亡友方能力，战术灵活

---

## 二、游戏目标

**胜利条件**：  
将对方所有弹珠击出棋盘或使其死亡。  
**失败条件**：  
己方所有弹珠死亡。

---

## 三、棋盘与坐标系统

### 3.1 棋盘结构
- 正六边形区域，每条边8个格点
- 总格点数：**169个**

### 3.2 坐标系统
- 中心为原点，两条120°夹角直线为x、y轴
- 格点坐标取整数，原点周围六点为：(1,0), (1,1), (0,1), (-1,0), (-1,-1), (0,-1)

### 3.3 边界判定
```
D = max(|x|, |y|, |x-y|)
D ≥ 8 → 出界
```

---

## 四、初始配置

### 4.1 阵营与位置
- 红方（左方） / 蓝方（右方）
- 红方初始区域：x ∈ [-7, -2]，D ≤ 8
- 蓝方初始区域：x ∈ [2, 7]，D ≤ 8

### 4.2 选珠阶段
- 每方拥有 **6个弹珠**
- 从6种颜色中任意选取（可重复）
- 双方自行排布阵型

### 4.3 先后手
随机决定红方或蓝方先行动

---

## 五、回合流程（网络同步）

1. 玩家选择一个己方存活弹珠
2. 根据弹珠颜色执行其专属移动指令
3. 结算所有碰撞、出界、死亡、特性触发（按物理顺序）
4. 检查胜负（若一方弹珠数量为0，游戏结束）
5. 切换回合，对方行动

> 随机结果由服务器生成并广播，保证一致性

---

## 六、六种弹珠详细设计

| 颜色  | 定位    | 移动方式         | 特殊能力                          |
| --- | ----- | ------------ | ----------------------------- |
| 白球  | 基础/继承 | 方向+力度1~5     | 被友方碰撞时+1剩余步数；友方其他色死亡时变色（黄球除外） |
| 蓝球  | 召唤/协同 | 方向+力度1~5     | 移动前生成2个随从（同方向同力度移动）；随从出界→蓝球死亡 |
| 绿球  | 推挤/控制 | 方向+力度1~5     | 移动后向远离方向推开相邻弹珠（可链式推挤1次）       |
| 红球  | 多段/灵活 | 步数1~4，每步可选方向 | 每步后立即碰撞并交换速度                  |
| 黑球  | 干扰/强制 | 不可主动移动       | 强制敌方弹珠移动（随机偏移±60°，步数2或3）      |
| 黄球  | 牺牲/增益 | 方向+力度1~5     | 死亡时强化友方其他弹珠（蓝/绿/红/黑）          |

### 6.1 黄球死亡增益效果

| 目标弹珠 | 增益效果 |
|----------|------------|
| 蓝球 | 随从出界不再导致蓝球死亡 |
| 绿球 | 推挤范围扩大（1 + 增益次数） |
| 红球 | 移动步数上限提升（4 + 增益次数） |
| 黑球 | 强制移动步数固定为3 |

---

## 七、碰撞与结算顺序

1. 弹珠**逐格推进**移动
2. 进入有弹珠的格点 → **弹性碰撞**（速度交换）
3. 每步结束后立即检查出界 → 出界即死亡
4. 绿球推挤在移动完全停止后**同时结算**
