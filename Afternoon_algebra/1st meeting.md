- [x] 在实际的代码合并过程中，虽然理论上可以分别修改再合并，但是最安全、最保险的手段一定是一个人进行修改之后立即提交到github上，之后另外的人在进行update之后自己在新的基础上拼装自己的新代码。这件事情如何协调？

1. **尽快合并其中一个人的 PR**，另一个人及时从最新的 `main` 拉取更新（`git fetch && git merge main` 或 rebase）。
    
2. **遇到冲突**在本地解决并测试后再提交。
    
3. **避免两人长期修改同一个文件**，即使行号不重叠，越长时间越容易产生隐藏冲突（格式调整、空行变化等）。
