# Marble3D 弹珠基类 —— 外部接口文档

## 一、概述

`Marble3D` 是所有弹珠（白、蓝、绿、红、黑、黄）的基类，实现了统一的移动、碰撞、出界、死亡逻辑。 

外部模块（游戏流程、棋盘管理、UI）通过以下接口与该类交互。

---

## 二、公共方法（外部主动调用）

### 1. `move(direction: int, steps: int) -> void`

**作用**：玩家回合主动移动弹珠。 

**参数**：

- `direction`：0～5，对应 `MarbleConst.HexDirection` 枚举顺序

- `steps`：1～5，移动步数

**行为**：

- 逐格移动，自动处理弹性碰撞、出界死亡

- 自动触发子类的特殊能力（蓝球随从、白球变色等）

**调用时机**：当前玩家选择己方存活弹珠后调用。

---

### 2. `die() -> void`

**作用**：强制弹珠死亡（出界或碰撞致死）。 

**行为**：

- `is_alive = false`

- 调用 `on_death()` 钩子（子类覆写可实现黄球增益等）

- 从棋盘移除节点并释放

**注意**：外部不要直接 `queue_free`，统一使用 `die()`。

---

### 3. `get_current_hex() -> Vector2`

**作用**：获取弹珠当前六边形坐标（轴向 `(q, r)`）。 

**返回值**：`Vector2(q, r)` 

---

### 4. `continue_move(steps: int, direction: int) -> void`

**作用**：碰撞后让弹珠继续移动（内部使用，外部一般不需要主动调用）。

---

## 三、公共属性（只读访问）

| 属性         | 类型                        | 说明                 |
| ---------- | ------------------------- | ------------------ |
| `is_alive` | `bool`                    | 存活状态（移动前必须检查）      |
| `color`    | `MarbleConst.MarbleColor` | 弹珠颜色，白球变色时会动态修改    |
| `camp`     | `MarbleConst.Camp`        | 阵营（`RED` / `BLUE`） |

---

## 四、棋盘管理器（HexGrid3D）必须实现的接口

`Marble3D` 内部依赖以下方法，请确保 `HexGrid3D.gd` 已实现：

```gdscript

func get_marble_hex(marble: Marble3D) -> Vector2

func is_out_of_bounds(q: int, r: int) -> bool

func get_marble_at(q: int, r: int) -> Marble3D

func move_marble(marble: Marble3D, from_hex: Vector2, to_hex: Vector2) -> void

func remove_marble_by_node(marble: Marble3D) -> void

```
