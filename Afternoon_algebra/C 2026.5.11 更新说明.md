# 六色弹珠：玩家操作流程实现文档

## 概述

本文档记录了在 Godot 4 项目中实现以下三个交互功能的全部更改：

1. **玩家操作流程**：点击己方弹珠 → 选方向 → 选力度/步数 → 执行
2. **显示当前回合方**（红方/蓝方）
3. **显示基本提示**（选中球、选择方向、选择力度等）

---

## 涉及文件及改动说明

### 1. `GameManager.gd` —— 游戏状态机核心

**改动**：在原有测试代码基础上，新增完整的状态机系统。

```gdscript
class_name GameManager
extends Node2D

# 状态枚举
enum TurnState {
    IDLE,               # 等待点击己方弹珠
    MARBLE_SELECTED,    # 已选中弹珠，等待选方向
    DIRECTION_SELECTED, # 已选方向，等待选力度
    EXECUTING           # 移动中，禁止操作
}
```

| 新增内容 | 作用 |
|----------|------|
| `TurnState` 枚举 | 定义操作流程的 4 个阶段 |
| `current_state` 变量 | 追踪当前处于哪个阶段 |
| `selected_marble` | 存储玩家选中的弹珠 |
| `selected_direction: int` | 存储玩家选择的方向（0~5） |
| `selected_power: int` | 存储玩家选择的力度（1~5） |
| `current_team` | 当前行动方（RED/BLUE） |
| `turn_number` | 回合计数 |
| `start_turn()` | 开始新回合，重置所有选择状态 |
| `select_marble(marble)` | **右侧检查器 → Node → Groups → 添加 "marbles"** |
| `select_direction(dir)` | 记录方向，进入"待选力度"状态 |
| `select_power(power)` | 记录力度，触发移动执行 |
| `execute_move()` | 调用弹珠的 `move()` 方法，完成后切换回合 |
| `cancel_selection()` | 取消当前选择，回到 IDLE 状态 |

**保留的原有功能**：
- `hex_grid` 和 `test_marble` 引用
- `_ready()` 中的初始化测试
- `_input()` 中的回车键测试

---

### 2. `InputHandler.gd` —— 鼠标/键盘输入处理

**改动**：新建文件，负责将玩家操作转换为 GameManager 的状态调用。

```gdscript
extends Node2D
var game_manager: GameManager
```

| 新增内容 | 作用 |
|----------|------|
| `_unhandled_input(event)` | 监听鼠标左键点击和右键取消 |
| `_handle_click(pos)` | 根据当前状态分发给对应处理函数 |
| `_try_select_marble(pos)` | IDLE 状态下，将屏幕坐标转为六边形坐标，查找并选中弹珠 |
| `_try_select_direction(pos)` | MARBLE_SELECTED 状态下，检测 6 个邻居格点哪个被点击 |
| `_try_select_power(pos)` | DIRECTION_SELECTED 状态下，目前用键盘 1~5 选力度（空函数体，键盘在 `_input` 补充） |
| `_input(event)` | 键盘数字键 1~5 触发 `select_power()` |

**关键逻辑**：
- 点击弹珠时，调用 `hex_grid.world_to_hex()` 将像素坐标转为六边形坐标
- 选方向时，遍历 6 个邻居格点，比对点击位置与邻居格点像素坐标的距离

### 3. `Marble2D.gd` —— 弹珠基类

**改动**：新增高亮功能，不影响原有变量名和注释。

| 新增内容 | 作用 |
|----------|------|
| `is_highlighted: bool` | 高亮状态标记 |
| `highlight()` | 选中时调用，将弹珠 modulate 变亮 ×1.5 |
| `unhighlight()` | 取消选中时调用，恢复原始颜色 |
| `_get_sprite_node()` | 兼容普通弹珠的 `Sprite` 节点和白球的 `SpriteWhite` 节点 |
| `_update_sprite_color()` 改用 `_get_sprite_node()` | 白球变色也能正确更新外观 |

**保留不变的原有内容**：
- 所有导出变量（`color`, `camp`）
- 所有运行时状态变量（`is_alive`, `remaining_steps`, `current_dir`, `hex_grid`, `hex_coord`, `sprite`）
- 所有移动逻辑函数（`move`, `continue_move`, `_move_step_by_step`）
- 所有钩子函数（`on_before_move`, `on_after_move`, `on_step_moved`, `on_collision_with`, `on_collision_as_target`）
- 死亡处理函数（`die`, `on_death`）
- 辅助函数（`get_current_hex`, `get_neighbor_hex`）

### 4. `HexGrid2D.gd` —— 棋盘控制器

**改动**：新增坐标反向转换函数，修复缩进错误。

| 新增/修复内容 | 作用 |
|---------------|------|
| `world_to_hex(world_pos)` | 将屏幕像素坐标转换回六边形坐标 (q, r)，供 InputHandler 点击检测使用 |
| 缩进修复 | 将 `get_marble_hex`、`get_marble_at`、`move_marble`、`remove_marble_by_node` 从 `remove_marble` 函数内部移出，成为独立函数 |

## 操作流程图

```
玩家点击己方弹珠
    ↓
[IDLE] → 调用 world_to_hex() 定位格子 → get_marble_at() 获取弹珠 → select_marble()
    ↓
弹珠高亮 + 进入 MARBLE_SELECTED 状态
    ↓
玩家点击相邻格子
    ↓
[MARBLE_SELECTED] → 遍历 6 个邻居 → 比对点击位置 → select_direction(dir)
    ↓
进入 DIRECTION_SELECTED 状态
    ↓
玩家按数字键 1~5
    ↓
[DIRECTION_SELECTED] → select_power(power) → execute_move()
    ↓
进入 EXECUTING 状态 → 弹珠.move() → 等待完成 → 切换回合 → 回到 IDLE
```

---

## 场景节点结构要求

```
Main (Node2D, 挂载 GameManager.gd)
├── HexGrid2D (挂载 HexGrid2D.gd)
├── Marble (挂载 Marble2D.gd / WhiteMarble.gd / BlueMarble.gd)
├── InputHandler (挂载 InputHandler.gd)
└── UI (CanvasLayer, 挂载 UI.gd)
    ├── TurnLabel (Label)
    └── MessageLabel (Label)
```

每个弹珠节点需要添加 Group `marbles`。
